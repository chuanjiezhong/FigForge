# 代码保护说明

## 当前保护措施

### 1. ASAR 打包
- Electron 默认将代码打包到 `app.asar` 文件中
- ASAR 是一种简单的归档格式，**可以被解压查看**
- 使用工具如 `asar` 可以轻松解压：`npx asar extract app.asar output/`

### 2. 代码混淆（已配置）
- 使用 `javascript-obfuscator` 对代码进行混淆
- 混淆后的代码：
  - 变量名和函数名被替换为无意义的名称
  - 代码结构被改变
  - 字符串被编码
  - 增加了解读难度

### 3. 代码压缩
- 使用 Terser 进行代码压缩
- 移除空白和注释
- 简化代码结构

## 保护级别

### ⚠️ 重要提示

**完全保护 JavaScript 代码是不可能的**，因为：
1. 代码必须在客户端执行
2. 浏览器/Electron 需要能够解析和执行代码
3. 有经验的开发者仍然可以：
   - 解压 asar 文件
   - 使用反混淆工具（虽然效果有限）
   - 通过调试工具分析代码

### 保护效果

- ✅ **防止普通用户**：普通用户无法轻易查看源码
- ✅ **增加难度**：增加了解读代码的难度和时间成本
- ⚠️ **专业开发者**：有经验的开发者仍然可以分析代码（但需要更多时间）

## 使用方式

### 开发环境（不混淆）
```bash
yarn build  # 正常构建，不混淆
```

### 生产环境（混淆）
```bash
yarn build:prod  # 构建并混淆代码
yarn dist:mac    # 打包应用
```

## 混淆配置

混淆配置在 `vite-plugin-obfuscator.ts` 中，可以根据需要调整：

- **compact**: 压缩代码
- **controlFlowFlattening**: 控制流扁平化
- **deadCodeInjection**: 注入死代码
- **selfDefending**: 自我保护（防止格式化）
- **stringArrayEncoding**: 字符串编码方式
- **debugProtection**: 调试保护（可能影响性能）

## 额外保护建议

### 1. 关键逻辑放在服务器端
- API 密钥、认证逻辑等放在服务器
- 客户端只负责 UI 和展示

### 2. 使用原生模块
- 将核心算法用 C++/Rust 编写
- 编译为原生模块，更难反编译

### 3. 代码分割
- 将关键代码延迟加载
- 减少一次性暴露的代码量

### 4. 许可证验证
- 添加许可证验证机制
- 防止未授权使用

### 5. 代码签名
- 对应用进行代码签名
- 防止被篡改

## 验证保护效果

打包后可以验证：

```bash
# 1. 解压 asar 文件
npx asar extract release/mac/FigForge.app/Contents/Resources/app.asar ./extracted

# 2. 查看代码（应该已经被混淆）
cat extracted/main/index.js | head -20
```

如果看到类似这样的代码，说明混淆成功：
```javascript
var _0x1234=['base64','decode',...];function _0x5678(_0x9abc,_0xdef0){...}
```

## 注意事项

1. **性能影响**：混淆会增加代码体积和执行时间（通常影响很小）
2. **调试困难**：混淆后的代码难以调试，建议开发时关闭混淆
3. **错误报告**：混淆后的错误堆栈难以阅读，需要 source map
4. **兼容性**：某些混淆选项可能影响代码执行，需要测试

## 总结

- ✅ 已配置代码混淆，增加代码保护
- ⚠️ 无法完全防止代码被查看，但可以显著增加难度
- 💡 关键逻辑建议放在服务器端或使用原生模块
