# 修复白屏问题

## 问题分析

打包后的应用显示白屏，可能的原因：

1. **Renderer 未构建**：`out/` 目录下没有 `renderer` 目录
2. **Renderer 未打包**：构建了但没有被打包到 `app.asar` 中
3. **路径错误**：主进程加载 renderer 的路径不正确
4. **资源加载失败**：HTML 中的脚本路径在打包后找不到

## 解决方案

### 1. 确保 Renderer 被构建

检查 `out/renderer/` 目录是否存在：

```bash
ls -la out/renderer/
```

如果不存在，需要修复构建问题。

### 2. 确保 Renderer 被打包

检查 `package.json` 中的 `files` 配置，确保包含：

```json
{
  "files": [
    "out/**/*",  // 这会包含 out/renderer/
    ...
  ]
}
```

### 3. 修复加载路径

已更新 `main/main.ts`，添加了多个备用路径尝试：

- `out/renderer/index.html` (相对路径)
- `app.asar/out/renderer/index.html` (asar 路径)
- `app.asar/renderer/index.html` (备用路径)
- `app.asar/index.html` (根路径)

### 4. 临时调试

已添加 DevTools 自动打开，可以在控制台查看错误信息。

## 快速修复步骤

1. **清理并重新构建**：
```bash
rm -rf out release
yarn build
```

2. **检查构建输出**：
```bash
ls -la out/renderer/
```

3. **如果 renderer 存在，重新打包**：
```bash
yarn dist:mac
```

4. **运行打包后的应用，查看控制台错误**

## 如果仍然白屏

1. 打开 DevTools（已自动打开）
2. 查看 Console 标签的错误信息
3. 查看 Network 标签，检查哪些资源加载失败
4. 根据错误信息调整路径配置

## 常见错误

### 错误：Failed to load resource
- **原因**：脚本路径不正确
- **解决**：检查 HTML 中的脚本路径，确保使用相对路径

### 错误：Cannot find module
- **原因**：模块路径不正确
- **解决**：检查 Vite 构建配置，确保路径别名正确

### 错误：Blank page
- **原因**：React 应用未正确挂载
- **解决**：检查 `main.tsx` 和 `App.tsx` 是否有错误
