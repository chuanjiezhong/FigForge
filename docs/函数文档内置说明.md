# 函数文档内置功能说明

## 功能概述

现在支持将R函数的文档和参数信息内置到项目中，这样用户就不需要每次都从R包中动态获取文档了。同时，参数配置界面也更加友好，会根据参数类型显示不同的输入控件。

## 使用方法

### 方式一：通过界面添加（推荐）

1. 点击工具栏中的"函数文档"按钮
2. 在打开的对话框中：
   - 查看现有函数文档列表
   - 点击"添加参数"按钮添加函数参数
   - 填写函数名、包名、描述、详细文档等信息
   - 为每个参数配置类型、描述、是否必需等信息
   - 点击"保存"按钮保存

### 方式二：直接编辑配置文件

配置文件位置：
- **开发环境**：项目根目录下的 `function-docs.json`
- **打包后**：用户数据目录下的 `function-docs.json`
  - macOS: `~/Library/Application Support/FigForge/function-docs.json`
  - Windows: `%APPDATA%/FigForge/function-docs.json`
  - Linux: `~/.config/FigForge/function-docs.json`

## 配置文件格式

参考 `function-docs.json.example` 文件，格式如下：

```json
{
  "functions": [
    {
      "name": "函数名",
      "package": "包名（可选）",
      "description": "函数简短描述",
      "documentation": "详细文档说明（支持多行）",
      "examples": "使用示例代码",
      "detailedParameters": [
        {
          "name": "参数名",
          "type": "参数类型",
          "description": "参数描述",
          "required": true/false,
          "default": "默认值",
          "placeholder": "占位符文本",
          "min": 最小值（仅number类型）,
          "max": 最大值（仅number类型）,
          "options": ["选项1", "选项2"]（仅select类型）
        }
      ],
      "version": "版本号（可选）",
      "author": "作者（可选）"
    }
  ]
}
```

## 参数类型说明

- `string`: 字符串输入框
- `number`: 数字输入框（可设置min/max）
- `boolean`: 开关按钮
- `array`: 多行文本输入（每行一个元素）
- `file`: 文件选择器
- `directory`: 目录选择器
- `select`: 下拉选择框（需要提供options数组）
- `text`: 多行文本输入框

## 功能特点

1. **优先使用内置文档**：如果函数有内置文档，会优先显示内置文档，否则才从R包获取
2. **智能参数输入**：根据参数类型自动显示合适的输入控件
3. **逐步添加**：可以一个函数一个函数地添加文档，不需要一次性完成
4. **向后兼容**：如果没有内置文档，仍然会从R包动态获取

## 注意事项

1. 函数名和包名的组合必须唯一
2. 参数名不能重复
3. 如果函数有内置文档，会优先使用内置文档，不会从R包获取
4. 配置文件使用UTF-8编码

## 如何添加函数文档

你可以按照以下步骤添加函数文档：

1. 准备函数信息：
   - 函数名
   - 包名（如果有）
   - 函数描述
   - 详细文档
   - 参数列表及每个参数的详细信息

2. 使用界面添加：
   - 打开"函数文档"管理界面
   - 填写函数信息
   - 逐个添加参数并配置
   - 保存

3. 或者直接编辑配置文件：
   - 复制 `function-docs.json.example` 为 `function-docs.json`
   - 按照格式添加函数信息
   - 保存文件

## 常见问题

**Q: 如何更新已存在的函数文档？**  
A: 在函数文档管理界面中，点击"编辑"按钮，修改后保存即可。

**Q: 如何删除函数文档？**  
A: 在函数文档管理界面中，点击"删除"按钮即可。

**Q: 内置文档和R包文档冲突怎么办？**  
A: 内置文档优先级更高，如果存在内置文档，会使用内置文档。

**Q: 可以批量导入函数文档吗？**  
A: 目前不支持批量导入，但可以直接编辑配置文件来批量添加。
