# Sharp 原生依赖打包说明

## 问题

Sharp 包含原生二进制文件，在 Electron 打包时需要特殊处理。如果配置不当，会出现以下错误：

```
Error: Could not load the "sharp" module using the darwin-arm64 runtime
ERR_DLOPEN_FAILED: dlopen(...): Library not loaded: @rpath/libvips-cpp.42.dylib
```

## 解决方案

### 1. electron-builder 配置

在 `package.json` 中已配置：

```json
{
  "build": {
    "asarUnpack": [
      "node_modules/sharp/**/*",
      "node_modules/@img/**/*"
    ],
    "files": [
      "out/**/*",
      "node_modules/**/*"
    ]
  }
}
```

### 2. 应用启动时设置路径

在 `main/main.ts` 中已添加路径设置代码。

### 3. 重新打包

```bash
# 清理旧的构建
rm -rf out release

# 重新构建
yarn build

# 重新打包
yarn dist:mac
```

## 验证

打包后，检查以下路径是否存在：

- `app.asar.unpacked/node_modules/sharp/`
- `app.asar.unpacked/node_modules/@img/`

如果这些路径存在，说明配置正确。

## 如果仍然有问题

1. **检查 sharp 版本**：确保使用兼容的 sharp 版本
2. **重新安装 sharp**：`yarn remove sharp && yarn add sharp`
3. **检查平台架构**：确保打包时使用正确的架构（arm64 或 x64）
4. **查看详细日志**：运行打包后的应用，查看控制台错误信息

## 替代方案

如果 sharp 仍然有问题，可以考虑：

1. **使用 Canvas API**：对于简单的图片处理，可以使用 Canvas
2. **使用其他库**：如 `jimp` 或 `canvas`（但功能可能不如 sharp 强大）
3. **延迟加载**：在需要时才加载 sharp，并提供降级方案
