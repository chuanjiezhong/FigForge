# React-Konva vs Fabric.js 详细对比

## 一、基本信息对比

| 特性 | React-Konva | Fabric.js |
|------|------------|-----------|
| **基础库** | Konva.js | Fabric.js (原生) |
| **框架绑定** | React 专用 | 框架无关（原生 JS） |
| **GitHub Stars** | ~5K+ | ~25K+ |
| **最新版本** | v18.x | v5.x |
| **许可证** | MIT | MIT |
| **维护状态** | 活跃 | 活跃 |
| **文档语言** | 英文为主 | 英文为主 |

## 二、核心架构对比

### React-Konva
**架构特点：**
- ✅ **React 封装**：基于 Konva.js 的 React 绑定
- ✅ **声明式 API**：使用 React 组件方式定义图形
- ✅ **响应式更新**：自动处理状态变化和重渲染
- ✅ **React 生态集成**：与 Redux、Zustand 等状态管理库无缝集成

**代码示例：**
```tsx
import { Stage, Layer, Rect, Circle } from 'react-konva';

function MyCanvas() {
  const [x, setX] = useState(50);
  
  return (
    <Stage width={800} height={600}>
      <Layer>
        <Rect x={x} y={50} width={100} height={100} fill="red" />
        <Circle x={200} y={200} radius={50} fill="blue" />
      </Layer>
    </Stage>
  );
}
```

### Fabric.js
**架构特点：**
- ✅ **原生 JavaScript**：不依赖任何框架
- ✅ **命令式 API**：直接操作对象
- ✅ **对象模型**：丰富的对象系统
- ✅ **框架无关**：可以在 React、Vue、Angular 中使用

**代码示例：**
```javascript
import { fabric } from 'fabric';

const canvas = new fabric.Canvas('canvas', {
  width: 800,
  height: 600
});

const rect = new fabric.Rect({
  left: 50,
  top: 50,
  width: 100,
  height: 100,
  fill: 'red'
});

canvas.add(rect);
```

## 三、功能特性对比

### React-Konva 功能

**核心功能：**
- ✅ **2D Canvas 渲染**：高性能 Canvas 操作
- ✅ **图形对象**：Rect, Circle, Line, Text, Image, Group 等
- ✅ **事件处理**：点击、拖拽、缩放等交互
- ✅ **动画系统**：流畅的动画和过渡
- ✅ **图层管理**：Layer 系统支持分层
- ✅ **变换操作**：旋转、缩放、平移
- ✅ **滤镜效果**：模糊、亮度、对比度等

**React 特性：**
- ✅ **组件化**：每个图形都是 React 组件
- ✅ **状态管理**：与 React state 无缝集成
- ✅ **生命周期**：支持 useEffect、useMemo 等 Hooks
- ✅ **性能优化**：支持 React.memo、useMemo 优化

**优势：**
- 声明式编程，符合 React 思维
- 自动处理重渲染优化
- 与 React 生态完美集成
- 学习曲线平缓（对 React 开发者）

### Fabric.js 功能

**核心功能：**
- ✅ **丰富的对象模型**：Rect, Circle, Triangle, Polygon, Path, Text, Image, Group 等
- ✅ **交互功能**：内置拖拽、缩放、旋转、选择框
- ✅ **SVG 支持**：SVG 解析和导出
- ✅ **序列化**：JSON 序列化/反序列化
- ✅ **历史记录**：Undo/Redo 功能
- ✅ **滤镜系统**：丰富的滤镜效果
- ✅ **文本编辑**：内置文本编辑功能
- ✅ **图像处理**：裁剪、滤镜、调整等

**高级功能：**
- ✅ **对象分组**：Group 和 ActiveSelection
- ✅ **对象锁定**：锁定位置和属性
- ✅ **对象克隆**：复制和粘贴
- ✅ **对齐辅助线**：自动对齐和吸附
- ✅ **自定义控件**：可自定义控制点
- ✅ **事件系统**：丰富的事件回调

**优势：**
- 功能极其丰富，开箱即用
- 内置编辑器功能（拖拽、选择等）
- SVG 支持完善
- 社区资源多，示例丰富

## 四、性能对比

### React-Konva 性能
**优势：**
- ✅ **高性能渲染**：基于 Konva.js，性能优秀
- ✅ **React 优化**：可以利用 React 的优化机制
- ✅ **按需渲染**：React 的虚拟 DOM 机制
- ✅ **动画流畅**：60fps 动画支持

**性能特点：**
- 适合中等复杂度的场景（< 1000 个对象）
- React 重渲染可能影响性能（需要优化）
- 大量对象时建议使用 React.memo

### Fabric.js 性能
**优势：**
- ✅ **原生性能**：直接操作 Canvas，无框架开销
- ✅ **对象缓存**：内置对象缓存机制
- ✅ **批量操作**：支持批量渲染优化

**性能特点：**
- 适合复杂场景（1000+ 对象）
- 原生 JS，性能可控
- 但功能丰富可能导致包体积较大

**性能建议：**
- **简单到中等复杂度**：React-Konva 足够
- **高复杂度（大量对象）**：Fabric.js 可能更好
- **需要精细性能控制**：Fabric.js 更灵活

## 六、在 React 中使用对比

### React-Konva 集成
**集成方式：**
```tsx
// 直接使用，无需额外封装
import { Stage, Layer, Rect } from 'react-konva';

function Editor() {
  return (
    <Stage width={800} height={600}>
      <Layer>
        <Rect x={50} y={50} width={100} height={100} />
      </Layer>
    </Stage>
  );
}
```

**优势：**
- ✅ 原生 React 组件，无缝集成
- ✅ 无需额外封装
- ✅ 状态管理自然

### Fabric.js 集成
**集成方式：**
```tsx
// 需要使用 useRef 和 useEffect
import { useEffect, useRef } from 'react';
import { fabric } from 'fabric';

function Editor() {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const fabricCanvasRef = useRef<fabric.Canvas | null>(null);

  useEffect(() => {
    if (canvasRef.current) {
      fabricCanvasRef.current = new fabric.Canvas(canvasRef.current, {
        width: 800,
        height: 600
      });
    }
    
    return () => {
      fabricCanvasRef.current?.dispose();
    };
  }, []);

  return <canvas ref={canvasRef} />;
}
```

**挑战：**
- ⚠️ 需要手动管理生命周期
- ⚠️ 需要手动同步 React 状态和 Fabric 对象
- ⚠️ 需要封装才能更好地与 React 集成

## 七、功能需求匹配度

### FigForge 排版编辑器需求

根据需求文档，排版编辑器需要：

1. ✅ **支持 SVG/PNG/TIFF 等多种 panel 类型**
   - React-Konva：✅ 支持 Image 组件加载各种格式
   - Fabric.js：✅ 支持 Image 对象，SVG 支持更好

2. ✅ **支持多 panel 画布（A/B/C…）**
   - React-Konva：✅ 使用 Group 或 Layer 管理
   - Fabric.js：✅ 使用 Group 或 Canvas 管理

3. ✅ **拖拽、缩放、裁剪、旋转**
   - React-Konva：✅ 需要自己实现拖拽逻辑
   - Fabric.js：✅ **内置拖拽、缩放、旋转功能**

4. ✅ **自动对齐、吸附、等间距**
   - React-Konva：⚠️ 需要自己实现
   - Fabric.js：✅ 有对齐辅助线，但需要自己实现吸附

5. ✅ **自动 panel 标号**
   - React-Konva：✅ 使用 Text 组件
   - Fabric.js：✅ 使用 Text 对象

6. ✅ **文本、箭头、比例尺等标注**
   - React-Konva：✅ 需要自己实现箭头、比例尺
   - Fabric.js：✅ 需要自己实现，但对象模型更灵活

7. ✅ **全局字体、字号、线宽控制**
   - React-Konva：✅ 通过主题或 Context
   - Fabric.js：✅ 通过对象属性批量设置

8. ✅ **保存与恢复排版状态（layout.json）**
   - React-Konva：✅ 序列化组件状态
   - Fabric.js：✅ **内置 toJSON/loadFromJSON**

9. ✅ **所有排版操作不触发重新计算**
   - React-Konva：✅ React 的优化机制
   - Fabric.js：✅ 原生操作，不涉及计算

## 八、实际使用场景对比

### React-Konva 适合的场景

5. ✅ **框架无关**：可能迁移到其他框架

**典型应用：**
- 在线设计工具（类似 Figma）
- 图形编辑器
- 图片标注工具
- 复杂的排版工具

### 混合方案（可选）

如果既需要 React 的声明式优势，又需要 Fabric.js 的功能：

**方案：Fabric.js + React 封装层**

```tsx
// 创建 FabricCanvas 组件封装 Fabric.js
function FabricCanvas({ panels, onUpdate }) {
  const canvasRef = useRef<HTMLCanvasElement>(null);
  const fabricRef = useRef<fabric.Canvas | null>(null);

  useEffect(() => {
    // 初始化 Fabric.js
    fabricRef.current = new fabric.Canvas(canvasRef.current);
    
    // 同步 React 状态到 Fabric
    panels.forEach(panel => {
      // 添加 panel 到画布
    });
  }, [panels]);

  // 监听 Fabric 事件，同步到 React
  useEffect(() => {
    const canvas = fabricRef.current;
    if (!canvas) return;

    const handleUpdate = () => {
      const objects = canvas.getObjects();
      onUpdate(objects.map(obj => obj.toJSON()));
    };

    canvas.on('object:modified', handleUpdate);
    canvas.on('object:added', handleUpdate);
    canvas.on('object:removed', handleUpdate);

    return () => {
      canvas.off('object:modified', handleUpdate);
      canvas.off('object:added', handleUpdate);
      canvas.off('object:removed', handleUpdate);
    };
  }, [onUpdate]);

  return <canvas ref={canvasRef} />;
}
```

### 实现缩放和旋转

**React-Konva：**
```tsx
// 需要自己实现变换逻辑
import { Transformer } from 'react-konva';

function SelectableRect({ shapeProps, isSelected, onSelect }) {
  const shapeRef = useRef();
  const trRef = useRef();

  useEffect(() => {
    if (isSelected && trRef.current) {
      trRef.current.nodes([shapeRef.current]);
      trRef.current.getLayer().batchDraw();
    }
  }, [isSelected]);

  return (
    <>
      <Rect
        ref={shapeRef}
        {...shapeProps}
        draggable
        onClick={onSelect}
        onTap={onSelect}
      />
      {isSelected && (
        <Transformer
          ref={trRef}
          boundBoxFunc={(oldBox, newBox) => {
            // 限制缩放
            if (newBox.width < 5 || newBox.height < 5) {
              return oldBox;
            }
            return newBox;
          }}
        />
      )}
    </>
  );
}
```
