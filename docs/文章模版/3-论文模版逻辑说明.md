# `3.docx` 论文模版逻辑说明

本文档说明 `docs/文章模版/3.docx` 的**篇章结构**与**图表（含热图）在叙述中的逻辑**，便于与 FigForge / OmicsFlow 转录组流程的自动解读稿对齐。  
`3.docx` 对应一篇 **Frontiers 风格的多组学论文示例**（主题：精神分裂症与氧化磷酸化相关 Hub 基因），方法学上与当前 Pipeline 中 **limma 差异分析 + pheatmap 热图** 等可对照。

---

## 1. 整体叙事逻辑（IMRaD + 验证链）

| 板块 | 作用 |
|------|------|
| **Abstract** | 一句话背景 → 方法概要（数据来源、DEG、ssGSEA、WGCNA、机器学习、动物与单细胞验证）→ 主要发现 → 结论。 |
| **Introduction** | 疾病背景 → 线粒体/OXPHOS 与疾病的文献缺口 → 本文要解决的科学问题。 |
| **Methods** | 分层写清：**数据**（GEO 等）→ **差异分析**（limma，log2 转换与标准化）→ **热图**（pheatmap）→ **WGCNA**（可选）→ **富集/ssGSEA** → **机器学习筛 Hub** → **独立集验证 / 实验 / snRNA-seq**。 |
| **Results** | 按 **Figure 1 → 2 → 3…** 顺序，每张图一段：**图题复述 + 关键统计量 + 生物学含义**。 |
| **Discussion** | 与既往研究比较 → 机制假说 → 局限性 → 展望。 |

**可迁移到「流程解读稿」的要点**：用户跑通 Pipeline 后，软件侧更关心 **Methods 中与本次运行一致的算法名、对比组、输出路径**，以及 **Results 中与热图/火山图对应的写法骨架**（见下方「热图」与 `pipeline-interpretation-snippet-*.md`）。

| 文中方法/图 | 与 OmicsFlow 常见步骤的对应 |

## 3. 热图（Heatmap）在 `3.docx` 中的逻辑（写作角度）

从文中 **Figure 1** 及相关 Methods 可抽象出下列**审稿人友好**的说明要素（已改写为与疾病无关的通用表述，写入 `pipeline-interpretation-snippet-zh.md` / `en`）：

1. **目的**：展示组间差异基因（或特征基因集）在样本上的**表达模式**是否分组一致。  
2. **构图**：行为基因、列为样本；常对表达量做 **行标准化（z-score / scale）**，使热图突出「相对高/低」而非绝对值。  
3. **颜色**：色阶表示标准化后的表达强弱（具体配色与软件图例一致）。  
4. **聚类（若启用）**：行/列聚类反映基因或样本表达谱相似性；若不聚类，则顺序多按差异统计量或给定基因列表。  
5. **与差异结果的关系**：热图基因集合通常来自同一套 limma 阈值（或 Top *N*），与火山图、差异表互为补充——热图重**模式**，火山图重**单基因显著性**。

文中另出现 **Figure 4K** 等「基因表达与 ATP 等表型关联」的热图，属于**分子—表型关联**类热图；若 Pipeline 未产生此类数据，解读稿中不必强行照抄。
