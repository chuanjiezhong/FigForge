# 添加本地目录功能说明

## 功能概述

"添加本地目录"功能允许你直接指定一个**已经存在于本地**的目录路径，让 FigForge 能够识别和使用该目录中的 R Pipeline 代码。

## 使用场景

### 场景 1：手动克隆的 GitHub 仓库

你已经通过命令行或其他方式手动克隆了 GitHub 仓库到本地：

```bash
# 你已经在终端中执行了
git clone https://github.com/username/your-r-repo.git ~/my-r-pipelines
```

现在你想让 FigForge 使用这个已经存在的目录，而不是让应用重新克隆：

1. 在设置界面中，点击"添加本地目录"
2. 输入路径：`~/my-r-pipelines` 或 `/Users/username/my-r-pipelines`
3. 点击"添加目录"

### 场景 2：本地开发的 R 代码

你有本地的 R 代码目录（不是从 GitHub 克隆的）：

```
~/my-local-pipelines/
├── pipeline1/
│   ├── main.R
│   └── pipeline.json
└── pipeline2/
    └── main.R
```

直接添加这个本地目录：

1. 输入路径：`~/my-local-pipelines`
2. 点击"添加目录"

### 场景 3：共享目录或网络路径

你的 R 代码在共享目录或网络路径上：

```
/Volumes/shared/r-pipelines/
或
\\server\shared\r-pipelines\
```

直接添加这个路径即可。

### 场景 4：使用 Git Submodule

如果你在项目中使用 Git Submodule：

```bash
git submodule add https://github.com/username/repo.git external-pipelines/repo
```

添加相对路径：`./external-pipelines/repo`

## 两种方式的区别

### 方式一：从 GitHub 克隆（自动）

**适用场景**：
- 你想让应用自动克隆仓库
- 你还没有本地副本
- 你想让应用管理仓库的更新

**操作**：
- 输入 GitHub URL
- 应用自动克隆到 `~/.figforge/external-pipelines/`

### 方式二：添加本地目录（手动）

**适用场景**：
- 你已经手动克隆了仓库
- 你有本地的 R 代码目录
- 你想使用特定位置的目录
- 你想自己管理 Git 操作

**操作**：
- 输入本地目录路径
- 应用直接使用该目录，不进行克隆

## 路径格式

支持以下路径格式：

1. **绝对路径**：
   ```
   /Users/username/r-pipelines
   C:\Users\username\r-pipelines
   ```

2. **相对路径**（相对于项目根目录）：
   ```
   ./external-pipelines/repo
   ../shared-pipelines
   ```

3. **使用 `~` 符号**（自动展开为用户主目录）：
   ```
   ~/r-pipelines
   ~/.figforge/pipelines
   ```

## 目录结构要求

本地目录必须包含符合 FigForge 要求的 Pipeline 结构：

```
your-directory/
├── pipeline1/          # 每个 pipeline 一个子目录
│   ├── main.R         # 必需：主 R 脚本
│   ├── pipeline.json  # 可选：Pipeline 信息
│   └── ...            # 其他依赖文件
├── pipeline2/
│   ├── main.R
│   └── ...
└── ...
```

## 示例

### 示例 1：添加手动克隆的仓库

```bash
# 1. 在终端中手动克隆
cd ~
git clone https://github.com/yourlab/seurat-pipelines.git my-pipelines

# 2. 在 FigForge 设置中添加
目录路径: ~/my-pipelines
```

### 示例 2：添加本地开发的代码

```
目录路径: ~/Documents/my-r-scripts
```

### 示例 3：添加 Git Submodule

```
目录路径: ./external-pipelines/your-repo
```

## 注意事项

1. **目录必须存在**：添加前确保目录已经存在
2. **权限要求**：确保应用有读取该目录的权限
3. **Git 管理**：如果目录是 Git 仓库，你可以自己用 `git pull` 更新，应用不会自动更新
4. **路径格式**：使用正确的路径格式，支持 `~` 展开

## 常见问题

### Q: 添加本地目录后，如何更新代码？

A: 如果目录是 Git 仓库，你可以：
- 在终端中执行 `git pull`
- 或者在应用的"已配置的外部目录"列表中点击"更新"按钮（如果支持）

### Q: 可以同时使用"克隆"和"添加本地目录"吗？

A: 可以！两种方式可以混合使用：
- 一些仓库通过应用自动克隆
- 一些目录手动添加

### Q: 添加的目录会被删除吗？

A: 不会。应用只是读取目录中的 Pipeline，不会删除或修改目录内容。

### Q: 相对路径是相对于哪里？

A: 相对于应用的工作目录（通常是应用安装目录或用户数据目录）。
