# 私有 GitHub 仓库配置说明

FigForge 支持克隆和更新私有 GitHub 仓库的 R 代码。对于私有仓库，需要配置认证信息。

## 认证方式

### 方式一：SSH 密钥（推荐）

适用于使用 SSH URL 的私有仓库。

#### 1. 生成 SSH 密钥（如果还没有）

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
# 或使用 RSA
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

#### 2. 将公钥添加到 GitHub

```bash
# 复制公钥内容
cat ~/.ssh/id_ed25519.pub
# 或
cat ~/.ssh/id_rsa.pub
```

然后：
1. 登录 GitHub
2. 进入 Settings > SSH and GPG keys
3. 点击 "New SSH key"
4. 粘贴公钥内容并保存

#### 3. 在 FigForge 中使用

在设置界面中：
- **仓库 URL**：使用 SSH 格式，如 `git@github.com:username/private-repo.git`
- **SSH 密钥路径**：
  - **留空**：Git 会自动使用默认密钥（`~/.ssh/id_rsa` 或 `~/.ssh/id_ed25519`），如果默认密钥已添加到 GitHub，则无需填写
  - **填写**：只有在需要使用非默认密钥时才需要填写，如 `~/.ssh/custom_key`

### 方式二：Personal Access Token (PAT)

适用于使用 HTTPS URL 的私有仓库。

#### 1. 创建 Personal Access Token

1. 登录 GitHub
2. 进入 Settings > Developer settings > Personal access tokens > Tokens (classic)
3. 点击 "Generate new token (classic)"
4. 设置权限：
   - 至少需要 `repo` 权限（完整仓库访问权限）
5. 生成并复制 token（格式：`ghp_xxxxxxxxxxxx`）

#### 2. 在 FigForge 中使用

在设置界面中：
- **仓库 URL**：使用 HTTPS 格式，如 `https://github.com/username/private-repo.git`
- **Personal Access Token**：输入刚才生成的 token

## 使用示例

### 示例 1：使用 SSH（推荐，最简单）

如果默认 SSH 密钥已添加到 GitHub：

```
仓库 URL: git@github.com:yourlab/private-pipelines.git
分支: main
SSH 密钥路径: （留空，使用默认密钥）
```

如果需要使用非默认密钥：

```
仓库 URL: git@github.com:yourlab/private-pipelines.git
分支: main
SSH 密钥路径: ~/.ssh/custom_key
```

### 示例 2：使用 Personal Access Token

```
仓库 URL: https://github.com/yourlab/private-pipelines.git
分支: main
Personal Access Token: ghp_xxxxxxxxxxxx
```

## 注意事项

1. **安全性**：
   - SSH 密钥路径会保存在配置文件中，但密钥文件本身不会被复制
   - Personal Access Token 会保存在配置文件中，请妥善保管
   - 建议使用 SSH 密钥方式，更安全

2. **URL 格式**：
   - SSH URL：`git@github.com:owner/repo.git`
   - HTTPS URL：`https://github.com/owner/repo.git`

3. **密钥路径**：
   - 支持绝对路径：`/Users/username/.ssh/id_rsa`
   - 支持 `~` 展开：`~/.ssh/id_rsa`

4. **权限要求**：
   - Personal Access Token 需要 `repo` 权限
   - SSH 密钥需要添加到 GitHub 账户

## 故障排查

### SSH 认证失败

**错误**：`Permission denied (publickey)`

**可能原因**：
1. SSH 密钥路径错误
2. 密钥未添加到 GitHub
3. 密钥权限不正确

**解决**：
1. 检查密钥路径是否正确
2. 确认公钥已添加到 GitHub
3. 检查密钥文件权限：`chmod 600 ~/.ssh/id_rsa`

### Token 认证失败

**错误**：`Authentication failed` 或 `Repository not found`

**可能原因**：
1. Token 已过期
2. Token 权限不足
3. Token 输入错误

**解决**：
1. 检查 Token 是否有效
2. 确认 Token 有 `repo` 权限
3. 重新生成 Token 并更新配置

### 混合使用

**注意**：不要同时配置 SSH 密钥和 Token。根据 URL 类型选择：
- SSH URL → 使用 SSH 密钥
- HTTPS URL → 使用 Token

## 配置文件示例

配置文件会自动保存认证信息：

```json
{
  "localPipelinesDir": "./pipelines",
  "externalPipelinesDirs": [
    "~/.figforge/external-pipelines/private-repo"
  ],
  "gitRepositories": [
    {
      "url": "git@github.com:yourlab/private-repo.git",
      "branch": "main",
      "localPath": "~/.figforge/external-pipelines/private-repo",
      "autoUpdate": false,
      "auth": {
        "sshKeyPath": "~/.ssh/id_rsa"
      }
    }
  ]
}
```

或使用 Token：

```json
{
  "gitRepositories": [
    {
      "url": "https://github.com/yourlab/private-repo.git",
      "branch": "main",
      "localPath": "~/.figforge/external-pipelines/private-repo",
      "auth": {
        "token": "ghp_xxxxxxxxxxxx"
      }
    }
  ]
}
```
